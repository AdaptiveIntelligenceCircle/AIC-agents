### POSITIONING.md

#### Why AIC-agents Exists

Most modern agent frameworks optimize for:

* Speed
* Scale
* Capability expansion

AIC-agents exists to optimize for:

* Safety over speed
* Responsibility over scale
* Trust over capability

---

#### Key Differentiators

| Conventional Agents  | AIC-agents            |
| -------------------- | --------------------- |
| Opaque reasoning     | Inspectable decisions |
| Irreversible actions | Built-in rollback     |
| Static permissions   | Trust-based authority |
| Scale-first          | Control-first         |

---

#### Audience

AIC-agents is designed for:

* Researchers
* System architects
* Security engineers
* Policymakers (technical)

It is *not* designed for quick demos or hype-driven products.

---

#### Position Statement

> If an agent cannot be stopped, it should not exist.

---

These documents define the non-negotiable foundations of AIC-agents.
